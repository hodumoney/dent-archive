# 치과 구인글 아카이브 (Dental Recruit Archive)

> **완전 무료로 웹사이트 만들기** → **[SETUP_GITHUB.md](SETUP_GITHUB.md)** 를 그대로 따라 하세요.
> (GitHub 에 올리면 하루 4번 자동으로 게시판을 확인해 웹페이지로 보여줍니다.)

---


dentphoto 「치과의사 구인구직양도」 게시판을 매일 수집해서 **삭제된 글을 보관**하고,
**올렸다 지웠다 반복하는 치과**를 자동으로 찾아내는 도구입니다.

- 게시글을 내 DB(SQLite)에 저장 — 원본이 삭제돼도 제목·본문이 남습니다.
- **삭제 감지**: 글 번호의 순번 공백(gap)으로, 뒤로 밀린 글과 실제로 지워진 글을 구분합니다.
- **반복 치과 식별**: **작성자(=치과 상호)** 기준으로 같은 치과의 글을 묶어,
  **몇 회째 재게시인지, 언제 올리고 언제 지웠는지**를 타임라인으로 보여줍니다.

---

## 1. 설치

```bash
pip install -r requirements.txt
```

## 2. 로그인 정보 확인

`secret.py` 에 아이디/비밀번호가 들어 있습니다 (이 파일은 `.gitignore` 처리되어 공유·커밋되지 않습니다).
환경변수 `DENT_ID` / `DENT_PW` 로 덮어쓸 수도 있습니다. **이 파일을 다른 사람과 공유하지 마세요.**

## 3. 화면 먼저 구경 (선택)

파서/로그인 튜닝 전에 대시보드 모습을 데모 데이터로 미리 볼 수 있습니다.

```bash
python seed_demo.py
python app.py          # http://127.0.0.1:5000
```

확인 후 실사용 전에는 반드시 데모 DB를 지우세요: `rm dent_tracker.db`

## 4. 로그인 + 파서 맞추기 (첫 실행에서 가장 중요)

```bash
python inspect_site.py
```

- **로그인 성공 여부**를 먼저 출력합니다.
  - 실패하면: `secret.py` 의 계정, 또는 `config.LOGIN_URL_CANDIDATES`(로그인 페이지 주소),
    또는 폼 필드명을 확인하세요. 로그인 폼 필드명은 브라우저 개발자도구(F12)로 로그인
    `<form>` 의 `<input name="...">` 을 보고 `config.LOGIN_ID_FIELD` / `LOGIN_PW_FIELD` 에 지정.
- 로그인 후 `debug_list.html`, `debug_view.html` 을 저장하고, 파서가 뽑은
  번호/구분/제목/작성자/날짜를 미리 보여줍니다.
- 목록 파싱이 어긋나면 `config.py` 의 `VIEW_LINK_KEYWORD`, `NO_QUERY_KEYS` 를 조정하세요.

> 인코딩(EUC-KR/CP949/UTF-8)은 `scraper.py` 가 자동 감지합니다.
> 상세글 링크는 목록의 실제 href 를 그대로 사용하므로 URL을 따로 맞출 필요가 없습니다.

## 5. 수집 실행

```bash
python crawl.py
```

로그인 → 앞쪽 `PAGES_TO_CRAWL`(기본 5)페이지 수집 → 신규 글 본문 보관 →
사라진 글 삭제 표시(기록 유지).

## 6. 대시보드 보기

```bash
python app.py
```

- `/` 반복 치과 랭킹 + 게시→삭제→재게시 타임라인
- `/posts` 전체/게시중/삭제됨 필터 + 제목·본문·작성자 검색, 구분(구인/구직/양도) 표시
- `/post/<번호>` 글 상세 (삭제된 글 본문도 보관되어 열람 가능) + 같은 치과의 다른 글

## 7. 매일 자동 실행 (cron)

수집을 자동화해야 반복·삭제 패턴이 쌓입니다. 빨리 지우는 치과를 놓치지 않으려면
하루 3~4회를 권장합니다.

```cron
# 매일 09,13,18,22시 수집 (경로는 실제 위치로 수정)
0 9,13,18,22 * * * cd /path/to/dent-tracker && /usr/bin/python3 crawl.py >> crawl.log 2>&1
```

대시보드를 상시 띄우려면:
```bash
pip install gunicorn && gunicorn -b 0.0.0.0:8000 app:app
```

## 동작 원리 & 한계

**삭제 감지 (gap 방식)** — 글 번호는 순차 증가합니다. 이번 수집에서 본 번호 범위
`[최소, 최대]` 안에 있는데도 안 보이는, 예전에 살아있던 글은 "구멍"이므로 삭제로 판정합니다.
새 글에 밀려 페이지 밖으로 내려간 것과 구분됩니다.
→ 한계: 살아있는 채로 수집 범위(`PAGES_TO_CRAWL`) 밖으로 내려간 뒤 지워진 글은 놓칠 수
   있습니다. 빨리 지우는 치과는 대개 글이 최신일 때 지우므로 실무상 잘 잡힙니다.

**치과 식별 (작성자 기준)** — 작성자 칸에 상호("서울오라클치과 김해점", "킹플란트치과")가
그대로 들어 있어, 작성자를 정규화해 식별키로 씁니다. 작성자가 비어 있으면 제목 속 상호,
그다음 전화번호, 제목 순으로 보조합니다. 같은 상호를 다른 지점이 쓰면 한 곳으로 합쳐질 수
있으니, 지점까지 구분하려면 작성자 원문을 그대로 키로 두면 됩니다(현재 방식).

**서버 예의 / 유의** — `REQUEST_DELAY_SEC`(기본 1.5초)로 간격을 둡니다. 상세글은 신규일 때
한 번만 받습니다. 수집·보관은 개인적 조사·참고 목적으로만 사용하고, 게시판 이용약관과
작성자·연락처 등 개인정보 취급에 유의하세요.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `secret.py` | 로그인 계정 (공유 금지, .gitignore) |
| `config.py` | 대상 사이트·로그인·수집 설정 (여기만 조정) |
| `inspect_site.py` | 로그인 확인 + 실제 HTML 저장 + 파서 결과 미리보기 |
| `scraper.py` | 로그인 세션 + HTTP + 한국어 인코딩 자동 처리 |
| `parser.py` | 목록/본문 파싱, 작성자 식별키 생성 |
| `db.py` | SQLite 스키마 및 저장 |
| `crawl.py` | 로그인+수집+삭제 감지 (매일 실행) |
| `analyze.py` | 반복 치과 그룹핑·타임라인 계산 |
| `app.py` + `templates/` | 대시보드 뷰어 |
| `seed_demo.py` | 데모 데이터 (미리보기용) |
